# AIO Insight Generation (AI視点での改善提案)

このスキルは、解析エンジンとスコアリングロジックの結果を受け取り、AI検索エンジンにとって「より魅力的なコンテンツ」に変えるための具体的なアクションを生成する手法を定義します。

## 1. 改善の 3 本柱

### A. リライト案 (Smart Rewriting)
- **逆ピラミッド構造**: AIが最初の数行で結論を把握できるように、重要な情報を冒頭に配置する。
- **要約用スニペット**: AIが「そのまま回答として抜粋できる」ブロック（TL;DR形式）の作成。

### B. トピックカバレッジの拡充
- **欠落トピックの特定**: 競合やナレッジグラフと比較して、ユーザーのQueryに対して不足している関連情報を特定。
- **FAQの自動精製**: ユーザーが検索しそうな質問に対する、具体的で引用しやすい回答案の作成。

### C. 信頼性の証拠補強
- **データ引用の推奨**: 「専門家によると」などの抽象的な表現を、「[機関名]の調査(202X年)ではX%が〜」といった具体的なデータに置き換える。

## 2. プロンプトエンジニアリングのベストプラクティス

改善案を生成する際の LLM プロンプトには以下の要素を含めること：

- **Role**: 「あなたは世界最高水準のAIOコンサルタントです。」
- **Input**: 対象ページのテキスト、現在のAIOスコア、ターゲットとするAIモデル（例：Perplexity）。
- **Format**:
  1. 現状の課題（なぜ引用されにくいか）
  2. 修正案（Before/After 形式）
  3. その修正による期待効果
- **Guidance**: 「GoogleのE-E-A-Tガイドラインと、AIの要約特性を両立させてください。」

## 3. 出力形式の設計

- **Markdown/HTML対応**: エンジニアやコンテンツ制作者がコピペで反映できる形式。
- **重要度（Priority）**: 改善効果が高い順にソートして提示。
